# AraratBot

Telegram-бот на aiogram 3.x для записи на экскурсию: подтверждение 18+, согласие на ПДн, выбор даты, контент о проекте и админ-рассылка.

## Возможности

- Главное меню: запись на экскурсию и раздел «О проекте»
- Юридический флоу: 18+, согласие на ПДн, опциональная рассылка
- Даты и лимит мест управляются через Google Sheets
- Контент (тексты и ссылки) управляется через лист `Content`
- Админ-команды: `/stats`, `/broadcast`

## Установка

```bash
pip install -r requirements.txt
```

Скопируйте `.env.example` в `.env` и заполните значения.

## Переменные окружения

| Переменная | Описание |
|---|---|
| `BOT_TOKEN` | Токен Telegram-бота от BotFather |
| `SPREADSHEET_ID` | ID Google Spreadsheet |
| `PRIVACY_POLICY_FILE` | Путь к PDF политики ПДн (по умолчанию `docs/Положение_о_политике_...pdf`) |
| `CONSENT_FORM_FILE` | Путь к PDF текста согласия (по умолчанию `docs/Soglasie_na_obrabotku_...pdf`) |
| `CONSENT_DOCUMENT_VERSION` | Версия документов для лога согласий (по умолчанию `1`) |
| `SERVICE_ACCOUNT_FILE` | Путь к JSON-ключу сервисного аккаунта Google |
| `ADMIN_IDS` | Telegram user_id администраторов через запятую |
| `CONSENT_LOG_FILE` | Локальный CSV-лог согласий ПДн (по умолчанию `consent_pd_log.csv`) |

## Google Sheets

Дайте доступ сервисному аккаунту к таблице (`Share` → email из `client_email` в JSON-ключе).

### Лист `Dates`

| A — дата | B — лимит мест (необязательно) |
|---|---|
| 2026-07-10 | 15 |
| 2026-07-12 | 20 |
| 2026-07-15 | |

Если столбец B пустой, лимит не применяется.

### Лист `Submissions`

Заполняется ботом автоматически:

| user_id | username | consent_pd | consent_mailing | chosen_date | name | phone | submission_timestamp |

### Лист `Content`

Материалы для раздела «О проекте»:

| A — заголовок | B — тип | C — содержимое |
|---|---|---|
| О маршруте | text | Текст о проекте... |
| Презентация | link | https://example.com/file.pdf |

Типы:
- `text` — текстовый блок в боте
- `link` — кнопка со ссылкой (PDF, сайт, Google Drive и т.д.)

### Лист `MailingList`

Заполняется ботом при согласии на рассылку:

| user_id | username | subscribed_at |

## Запуск

```bash
python bot.py
```

## Админ-команды

Доступны только пользователям из `ADMIN_IDS`.

- `/stats` — сколько записей на каждую дату
- `/broadcast` — рассылка подписчикам из `MailingList` (поддерживается HTML)
- `/cancel` — отмена текущего действия

## Проверка

1. `/start` → главное меню
2. «Записаться на экскурсию» → 18+ → политика и согласие (PDF в чате) → дата → имя → телефон
3. «О проекте» → материалы из листа `Content`

## Важно

- Не коммитьте `credentials.json`, `.env` и другие JSON-ключи Google в git.
- `MemoryStorage` теряет состояние после перезапуска бота. Для продакшена лучше Redis.
- Если ключ сервисного аккаунта попал в репозиторий — перевыпустите его в Google Cloud.
